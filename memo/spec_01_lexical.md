# SRU 言語仕様 — 字句構造

ソース: `csru/parser_constants.cc`, `csru/parser_spacing.cc`, `csru/sru_string.cc`。

SRU の字句解析はパーサジェネレータが組んだ手続きの中で実行される（独立した lexer は無い）。
以下は字句要素を実装ベースで整理したもの。

## 1. 文字集合

- ソースは ASCII / UTF-8 のバイト列として扱われる。
- パーサは入力を `string` として保持し、`int pos` をバイト単位で進める。
- マルチバイト識別子はサポートしない（識別子は ASCII 英数記号のみ）。

## 2. 空白とコメント

| トークン | 受理する文字 | 実装位置 |
| --- | --- | --- |
| `spc` | ` `, `\t` | `parser_spacing.cc:18` |
| `spc_or_comment` | `spc` + `#`〜行末 (`\n` を除く) | `parser_spacing.cc:33` |
| `spc_or_lf` | ` `, `\t`, `\r`, `\n` + `#`〜`\n`（`\n` 含む） | `parser_spacing.cc:59` |
| `lf` | `\r\n`、`\r`、または `\n` | `parser_spacing.cc:87` |
| `eos` | ソースの末尾 | `parser_spacing.cc:100` |

- コメントは `#` から行末まで（行頭・行末どちらでも書ける）。
- `spc_or_comment` はコメント本文と末尾の `\n` を消費するが、`\n` 自体は読み飛ばさない（statement の区切りに使う）。
- `spc_or_lf` は `#` 形式コメントを含む全空白を読み飛ばす。

## 3. 識別子 `ident`

`parser_constants.cc:22` `DEFINE_SRU_PROC(ident)`。

- 受理する文字: `[a-zA-Z0-9_!]`。1 文字目に数字単独は来ない（先頭が数字なら走査開始位置のままなので失敗扱い）。
- `!` も識別子の構成文字。ただし「`!=`」が次に現れたらそこで切る（`==` 等の比較演算子との衝突回避）。
- 最大マッチ。
- 推奨命名規則（`memo/naming.txt`）:
  - `ClassName`     : クラス名（大文字始まり キャメル）
  - `CONSTANT_NAME` : 定数
  - `valiableName`  : 変数・メソッド名（小文字始まり キャメル）
  - `_camelCase`    : クラスシステム内部用
  - `__camelCase`   : SRU システム内部用

## 4. 数値リテラル

### 整数 `number`
`parser_constants.cc:80`。
- 文字種: `[0-9]+`。
- 符号は字句に含まれない（先頭の `-` は構文上の単項マイナスとして扱う、`spec_02_grammar.md` 参照）。
- パース時には `Numeric.parse("...")` 呼び出しの AST に変換される。

### 実数 `real`
`parser_constants.cc:101`。
- 文字種: `[0-9.]+`。
- 小数点 `.` は最大 1 個。末尾が `.` なら失敗（`123.` は不可、`123.0` は可）。
- 指数表記 `e` / `E` は **未サポート**。
- 16 進・8 進・2 進リテラルは **未サポート**。
- 同じく `Numeric.parse("...")` 呼び出しの AST に変換される。

実装上、`real` パーサが `number` より先に試され、小数点を含む列は実数として処理される。

## 5. 文字列リテラル `const_string`

`parser_constants.cc:56`。
- 開始/終了は `"`。シングルクォート文字列は無い。
- 1 文字目が `"` でなければ失敗。
- 終了 `"` は「直前のバイトが `\\` でない最初の `"`」。複数行リテラルも `\n` を含めて許容される。
- `csru/sru_string.cc:202` の `UnEscapeString` がエスケープを処理。サポートされるシーケンス:
  - `\n` → LF (`0x0A`)
  - `\r` → CR (`0x0D`)
  - `\\` → `\`
  - `\"` → `"`
  - その他 `\X` は `X` そのもの（例: `\t` は `t` になる。タブを書きたければ生の TAB を埋める）
- `EscapeString` は `Inspect` 表示のときに `\n`, `\r`, `"`, `'` をバックスラッシュ付きに戻す。

## 6. 配列リテラル

`[` と `]` で囲む。中身は `expression` の `,` 区切り。末尾カンマ可。
詳細は `spec_02_grammar.md` の `array_literal` を参照。

## 7. クロージャ／ブロックリテラル

`{` と `}` で囲む。`|...|` 形式で仮引数列を、`:retVar` で継続用変数を指定できる。
詳細は `spec_02_grammar.md` を参照。

## 8. 文区切り

- 改行 `\n` (`lf`) または `;` または `eos` で文が区切られる。
- `then` キーワードは `if` の条件と本体を区切るための代替手段として認められる。
- カッコ内・配列リテラル内・closure 内では `spc_or_lf` が広く許され、改行で文を切ったことにはならない。

## 9. 予約語

文法トークンとして特別扱いされ、識別子としては使えない:

```
if    elsif    else    then    end
while
def
class
```

上記 9 語のみ。`true` / `false` / `nil` / `return` 等は **予約語ではなく** グローバル束縛された通常の識別子である（後述）。

## 10. 演算子トークン一覧

```
||  &&
==  ===  !=  >  <  >=  <=
|   &   <<  >>
+   -   *   /   %
!   ~
.   ,   ;   :
=
(   )   [   ]   {   }
|   (closure varg 区切り)
```

`===` は `==` より長いので長さ優先で受理する（`comp_super_equal` 規則。実装は等価チェックではあるが文法上は別トークン）。
